
# Ghid complet despre GitHub

## Ce este GitHub?
GitHub este o platformă de găzduire pentru controlul versiunilor și colaborare, care folosește sistemul Git. 
Este utilizată de dezvoltatori și echipe pentru a lucra împreună la proiecte software. 
Pe lângă controlul versiunilor, GitHub oferă funcționalități precum issue tracking, wiki, CI/CD, 
și un spațiu pentru colaborarea comunității.

---

## Terminologie importantă
1. **Repository (Repo)**: Un proiect sau folder care conține fișierele și istoricul versiunilor codului.
2. **Commit**: O înregistrare a modificărilor făcute asupra codului.
3. **Branch**: O ramură a proiectului pentru a lucra pe funcționalități sau bug-uri fără a afecta codul principal.
4. **Merge**: Procesul de a combina modificările dintr-o ramură în alta.
5. **Pull Request**: O propunere de modificare, utilizată pentru a revizui și aproba codul înainte de a-l integra.
6. **Fork**: O copie a unui repository public, folosită pentru a lucra independent.
7. **Clone**: Descărcarea unui repository pe calculatorul tău.
8. **Remote**: O versiune a repository-ului găzduită pe GitHub.
9. **Push/Pull**: 
    - **Push**: Trimiterea modificărilor din calculatorul tău către GitHub.
    - **Pull**: Preluarea modificărilor din GitHub în calculatorul tău.

## Cum să începi să folosești GitHub
1. **Creează-ți un cont**:
   - Accesează [github.com](https://github.com).
   - Creează un cont cu e-mailul și parola ta.

2. **Instalează Git** (dacă nu este instalat deja):
   - Descarcă Git de pe [git-scm.com](https://git-scm.com/).
   - Urmează instrucțiunile de instalare.

3. **Configurare inițială Git**:
   ```bash
   git config --global user.name "Numele Tau"
   git config --global user.email "emailul_tău@example.com"
   ```

4. **Creează un Repository pe GitHub**:
   - Apasă pe butonul „New” din dashboard-ul GitHub.
   - Completează numele proiectului, descrierea (opțional) și alege dacă este privat sau public.

5. **Clonează repository-ul local**:
   ```bash
   git clone https://github.com/nume-utilizator/nume-repository.git
   ```

6. **Adaugă fișiere și creează un commit**:
   ```bash
   git add .
   git commit -m "Mesaj despre modificările făcute"
   ```

7. **Trimite modificările pe GitHub**:
   ```bash
   git push origin main
   ```

8. **Creează și lucrează pe ramuri**:
   ```bash
   git checkout -b nume-ramura
   # Fă modificări
   git add .
   git commit -m "Mesaj pentru ramură"
   git push origin nume-ramura
   ```

9. **Fă un Pull Request**:
   - Mergi pe GitHub și apasă pe „Pull Requests”.
   - Creează o cerere pentru a integra modificările.

## Alte concepte importante

### 1. **SSH Keys**
Pentru a comunica cu GitHub fără a introduce parola de fiecare dată, poți folosi o cheie SSH.
- **Generare cheie SSH**:
  ```bash
  ssh-keygen -t ed25519 -C "emailul_tău@example.com"
  ```
  Alege locația implicită (~/.ssh/id_ed25519) și adaugă cheia în GitHub.
- **Adăugare cheie în GitHub**:
  - Mergi la *Settings* -> *SSH and GPG Keys* -> *New SSH Key*.
  - Introdu cheia generată din `cat ~/.ssh/id_ed25519.pub`.

### 2. **Git Ignore**
Fișierul `.gitignore` definește ce fișiere/foldere să fie ignorate de Git.
Exemplu de fișier `.gitignore`:
```
# Ignoră fișiere temporare
*.log
*.tmp

# Ignoră foldere
node_modules/
__pycache__/
```

### 3. **GitHub CLI**
GitHub CLI (`gh`) este un tool care îți permite să interacționezi cu GitHub din terminal.
- **Instalare**:
  - Urmează ghidul de pe [GitHub CLI](https://cli.github.com/).
- **Comenzi utile**:
  ```bash
  gh repo create nume-repository --public
  gh issue list
  gh pr create --fill
  ```

### 4. **Git Rebase vs Merge**
- **Rebase**: Rescrie istoricul commit-urilor pentru a crea o linie de timp curată.
- **Merge**: Creează un commit care combină ramurile.
  - Rebase este util pentru ramuri private.
  - Merge este preferat pentru lucrul colaborativ.

## Cele mai bune practici

1. **Scrie mesaje de commit clare**: Exemplu: `git commit -m "Fix: Rezolvat bug-ul X din funcția Y"`.
2. **Folosește ramuri pentru dezvoltare**: Creează o ramură nouă pentru fiecare funcționalitate sau bug.
3. **Revizuirea codului**: Utilizează Pull Requests pentru a permite colegilor să verifice modificările.
4. **Documentează proiectul**: Include un fișier `README.md` pentru a explica scopul proiectului și cum poate fi folosit.
5. **Testează înainte de a integra**: Rulează teste locale și folosește GitHub Actions pentru CI/CD.

## Comenzi Git avansate

1. **Resetare fișier**:
   - Revino la ultima versiune a unui fișier:
     ```bash
     git checkout -- nume-fișier
     ```

2. **Anulare ultimul commit** (fără a pierde modificările):
   ```bash
   git reset --soft HEAD~1
   ```

3. **Găsește eroarea cu bisect**:
   - Identifică commit-ul care a introdus un bug:
     ```bash
     git bisect start
     git bisect bad HEAD
     git bisect good nume-commit
     ```

4. **Vizualizare istoric**:
   ```bash
   git log --oneline --graph --decorate
   ```
